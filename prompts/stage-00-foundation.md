# Prompt — Stage 0: Repository foundation

Продолжи разработку проекта PulseOps в текущей рабочей папке.

Это отдельный чат только для **Stage 0 — Repository foundation**. Не начинай
Stage 1 в этом чате, даже если останется время.

## Обязательный контекст

Перед любыми изменениями полностью прочитай:

1. `AGENTS.md`;
2. `PRODUCT.md`;
3. `DEVELOPMENT_PLAN.md`;
4. `PROJECT_STATUS.md`.

Затем проверь содержимое папки и текущее состояние Git. Сохрани все существующие
планировочные документы.

## Цель этапа

Подготовить чистый, воспроизводимый фундамент React/TypeScript-приложения, на
котором можно безопасно выполнять следующие этапы.

## Выполни

1. Инициализируй в корне Vite-проект с React и TypeScript, не удаляя существующие
   `.md`-файлы и папку `prompts`.
2. Если Git-репозиторий фактически не инициализирован, корректно инициализируй
   его. Не создавай remote, commit или pull request.
3. Используй npm и создай актуальный lock-файл.
4. Установи согласованные основные зависимости:
   - React Router;
   - Redux Toolkit и React Redux;
   - Zod;
   - Recharts;
   - Lucide React;
   - date-fns.
5. Установи и настрой инструменты разработки:
   - ESLint;
   - Prettier;
   - Vitest;
   - React Testing Library;
   - jsdom;
   - MSW;
   - Playwright допускается установить сейчас, но браузеры пока не скачивать,
     если это требует лишнего времени или отдельного разрешения.
6. Включи строгий TypeScript. Не ослабляй стандартные strict-проверки.
7. Создай минимальную структуру:

```text
src/
  app/
    providers/
    router/
    store/
    styles/
  pages/
  features/
  entities/
  shared/
    api/
    config/
    lib/
    ui/
```

8. Удали демонстрационный контент Vite и сделай минимальный экран-заглушку
   PulseOps без реализации дизайн-системы или dashboard.
9. Добавь минимальный smoke test, подтверждающий рендер приложения.
10. Настрой npm scripts:
    - `dev`;
    - `build`;
    - `typecheck`;
    - `lint`;
    - `format`;
    - `format:check`;
    - `test`;
    - `test:run`;
    - `test:e2e` — может быть заготовкой для следующей настройки, но не должен
      ложно сообщать об успешном E2E-прогоне без тестов.
11. Добавь корректный `.gitignore`.
12. Создай минимальный `README.md` с назначением проекта, требованиями и
    командами локального запуска/проверки.

## Не делай

- не реализуй landing page, AppShell, Redux store, RTK Query или MSW handlers;
- не добавляй React Bits;
- не создавай backend и Cloudflare-конфигурацию;
- не меняй утверждённый scope;
- не добавляй Tailwind, Next.js, UI-kit или дополнительные библиотеки;
- не выполняй Stage 1.

## Обязательная проверка

Запусти и добейся успешного результата:

```text
npm run typecheck
npm run lint
npm run format:check
npm run test:run
npm run build
```

Также кратко запусти приложение локально и убедись, что страница открывается без
ошибок в консоли. Не оставляй фоновый dev-сервер после проверки.

## Завершение этапа

Перед финальным ответом обнови `PROJECT_STATUS.md` строго по разделу
`Required completion update`:

- Stage 0 можно отметить `complete` только после всех проверок;
- активным этапом установи `1 — Design tokens, routes and AppShell`;
- `Active prompt` установи в `not_created`;
- внеси точные команды и результаты проверок;
- обнови implementation summary, риски и stage history.

Если во время работы было принято решение, влияющее на следующие этапы, добавь
его в `Decision log` файла `DEVELOPMENT_PLAN.md`.

В финальном ответе укажи:

- что создано;
- какие проверки прошли;
- какие версии основного стека установлены;
- есть ли риски или блокеры;
- подтверждение, что Stage 1 не начинался.
